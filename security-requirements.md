## Claude Code Prompt — paste directly into the terminal session

Fix the following security issues in `app.py`, in priority order.
Use type hints on all function signatures and PEP 8 formatting throughout.
Do not add logging, new endpoints, or any changes unrelated to the fixes below.

### 1. HIGH — Disable debug mode
- `app.run(debug=True, ...)` must become `app.run(debug=False, ...)`
- Reason: debug mode exposes an interactive debugger to any caller in a shared environment

### 2. HIGH — Add Authorization header check to POST /variants
- Before processing the request body, check for an `Authorization` header
- If missing: return `401` with `{"error": "Authorization header required"}`
- Reason: write endpoint is completely unauthenticated — any caller can create records

### 3. MEDIUM — Add input validation to POST /variants
- Validate that the request body is valid JSON
- Check all required fields exist: `gene` (str), `chrom` (str), `pos` (int), `ref` (str), `alt` (str), `qual` (float)
- If invalid or missing: return `400` with `{"error": "<description>"}`
- Change response status from 200 to `201` for successful creation
- Reason: malformed input currently crashes the endpoint or silently stores bad data

### 4. MEDIUM — Fix GET /variants/`<id>` — match by id field, not array index
- Replace `variants[int(id)]` with a lookup that matches `v["id"] == int(id)` across the list
- Return `404` with `{"error": "Variant not found"}` if no match
- Return `400` if `id` is not a valid integer
- Reason: index-based lookup raises unhandled `IndexError` that exposes a stack trace

### 5. LOW — Fix case-sensitive gene search in search_variants
- Replace `v["gene"] == gene` with a case-insensitive comparison: `v["gene"].lower() == gene.lower()`
- Reason: `brca1` currently returns no results when stored value is `BRCA1`
